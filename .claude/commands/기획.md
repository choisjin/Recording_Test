새 기능 또는 변경사항의 구현 계획을 수립합니다. 코드를 수정하지 않습니다.

요청: $ARGUMENTS

## 자동 실행

### 1단계: 요구사항 정리
$ARGUMENTS에서 핵심 요구사항을 추출합니다.
--resume이면 HANDOFF.md를 읽고 이전 작업을 이어받습니다.

### 2단계: 코드베이스 탐색
Agent 도구(Explore)로 관련 코드를 탐색합니다:
- 유사 기능이 이미 구현되어 있는지 확인
- 변경이 필요한 파일 식별
- 의존 관계 파악
- 재사용 가능한 기존 함수/유틸리티 확인

### 3단계: 구현 계획 작성

```
## 구현 계획

**목표**: 한 줄 요약

**영향 파일**:
- 수정: [파일 경로] — [변경 내용]
- 신규: [파일 경로] — [역할]

**작업 단계** (순서대로):
1. [단계 제목]
   - 파일: [경로]
   - 작업: [구체적 변경 내용]

**참고 패턴**: 기존 코드에서 따를 패턴
**주의사항**: 잠재적 문제, 호환성 이슈
**예상 작업량**: 소/중/대
```

### 4단계: 다음 단계 안내
- 계획이 확정되면 `/auto`로 자동 실행하거나 직접 구현을 시작하세요
- 대규모 작업이면 HANDOFF.md 생성을 제안합니다

## 규칙
- 계획만 수립하고 코드를 수정하지 않습니다
- 각 단계는 하나의 커밋 단위가 될 수 있도록 분할합니다
- 5개 파일 이상 변경 시 반드시 단계별 분할합니다
