아키텍처 및 설계 결정에 대해 자문합니다. 코드를 수정하지 않습니다.

질문: $ARGUMENTS

## 자동 실행

### 1단계: 관련 코드 탐색
질문과 관련된 코드를 Agent 도구(Explore)로 탐색합니다:
- backend/app/services/ — 비즈니스 로직
- backend/app/routers/ — API 엔드포인트
- backend/app/plugins/ — 플러그인
- backend/app/models/ — 데이터 모델
- frontend/src/ — 프론트엔드

### 2단계: 분석
다음 구조로 답변합니다:

**현재 상태**: 지금 어떻게 되어 있는지 (관련 파일과 코드 위치 명시)

**선택지** (2-3가지):
각각의 장단점, 구현 난이도(소/중/대), 영향 파일

**추천**: 이 프로젝트에 가장 적합한 방법과 이유

**영향 범위**: 변경 시 영향 받는 파일 목록

## 규칙
- 코드를 절대 수정하지 않습니다
- "정석적 방법"보다 "이 프로젝트에 실용적인 방법"을 우선합니다
- 테스트 프레임워크가 없는 현실을 감안합니다
- 구현이 필요하면 `/auto` 또는 `/기획`을 안내합니다
