빌드, 배포, 서버 관리를 자동으로 수행합니다.

명령: $ARGUMENTS (상태|빌드|의존성|배포준비)

## 명령별 자동 실행

### "상태" — 환경 상태 종합 리포트
다음을 자동으로 확인하고 표로 보고합니다:
1. 포트 확인: `netstat -ano | findstr :8000` / `:5173`
2. 서버 응답: `curl -s http://localhost:8000/api/health`
3. Python: `python --version`
4. Node: `node --version`
5. Git: `git status --short` + 마지막 커밋
6. 디스크: scenarios, screenshots, results 디렉토리 파일 수

### "빌드" — 프로덕션 빌드
1. Python 구문 체크 (backend/app/ 전체)
2. TypeScript 타입 체크: `cd frontend && npx tsc --noEmit`
3. 프론트엔드 빌드: `cd frontend && npm run build`
4. 결과 확인: frontend/dist/ 존재 및 크기

### "의존성" — 의존성 관리
1. Python: `pip list --outdated` (상위 10개)
2. Node: `cd frontend && npm outdated`
3. requirements.txt와 실제 설치 버전 비교
4. 특정 패키지 지정 시: `pip show <package>`

### "배포준비" — 배포 전 체크리스트
자동 실행 후 결과를 체크리스트로 보고:
- [ ] git status clean
- [ ] Python 구문 검증 통과
- [ ] TypeScript 타입 체크 통과
- [ ] 프론트엔드 빌드 성공
- [ ] requirements.txt 동기화

모든 항목 통과 시 `deploy.bat` 실행 여부를 사용자에게 확인합니다.

## 규칙
- 서버 시작/종료/배포는 반드시 사용자 확인 후 실행
- 결과는 항상 표 형식으로 정리
- 인자 없이 호출하면 "상태"를 실행
