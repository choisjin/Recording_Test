변경된 코드의 품질을 자동으로 검증합니다.

인자: $ARGUMENTS (선택: 특정 파일 경로 또는 --full)

## 자동 실행

### 1단계: 변경 범위 파악
`git diff --name-only`와 `git diff --cached --name-only`로 변경 파일 목록을 확인합니다.
- $ARGUMENTS가 특정 파일이면 해당 파일만 검증
- --full이면 backend/app/ 전체 검증
- 인자 없으면 변경 파일만 검증

### 2단계: Python 구문 검증
변경된 `.py` 파일 각각에 대해 실행:
```
python -c "import ast; ast.parse(open('<file>', encoding='utf-8').read())"
```

### 3단계: TypeScript 타입 체크
변경된 `.ts` 또는 `.tsx` 파일이 있으면 실행:
```
cd frontend && npx tsc --noEmit --pretty
```

### 4단계: 패턴 정합성 검사
- **모델 변경**: models/scenario.py 변경 → frontend/src/services/api.ts에서 해당 필드 사용처 확인
- **라우터 변경**: routers/*.py 변경 → 프론트엔드 API 경로/파라미터 일치 확인
- **의존성 추가**: 새 import → backend/requirements.txt 또는 package.json 등록 확인
- **플러그인 변경**: 클래스명 == 파일명 규칙 확인, Connect/Disconnect 메서드 존재 확인
- **i18n**: UI 텍스트 하드코딩 → translations.ts 사용 여부

### 5단계: 보안 체크
- .env, credentials, 토큰, 비밀번호가 코드에 하드코딩되지 않았는지 확인
- git diff에서 민감 정보 노출 여부 스캔

### 6단계: 결과 리포트
각 단계 결과를 표로 요약:

| 항목 | 결과 | 상세 |
|------|------|------|
| Python 구문 | PASS/FAIL | ... |
| TypeScript | PASS/FAIL/SKIP | ... |
| 패턴 정합성 | PASS/WARN | ... |
| 보안 | PASS/WARN | ... |

모든 검증 통과 시 `/커밋`을 안내합니다.

## 규칙
- 코드를 수정하지 않습니다 — 검증 결과만 보고합니다
- 변경 파일이 없으면 "변경 파일 없음" 리포트를 출력합니다
