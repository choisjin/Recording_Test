런타임 오류 및 연결 문제를 자동으로 진단합니다.

증상: $ARGUMENTS

## 자동 실행

### 1단계: 증상 분류
$ARGUMENTS에서 문제 유형을 자동 분류:
- **연결 문제**: ADB, HKMC6th, Serial, VisionCamera
- **API 오류**: 400, 404, 500 응답
- **WebSocket 문제**: 화면 스트리밍 끊김, 재생 결과 미수신
- **이미지 비교**: SSIM 점수 이상, 템플릿 매칭 실패
- **빌드/실행**: 서버 시작 실패, 모듈 로딩 오류

### 2단계: 환경 확인
자동으로 실행:
- 서버 응답: `curl -s http://localhost:8000/api/health`
- 포트: `netstat -ano | findstr :8000`
- 프로세스: `tasklist | findstr python`

### 3단계: 유형별 코드 추적

**연결 문제:**
- ADB → backend/app/services/adb_service.py
- HKMC6th → backend/app/services/hkmc6th_service.py
- Serial → backend/app/plugins/SerialPlugin.py
- VisionCamera → backend/app/plugins/VisionCamera.py + VisionCameraClient.py
- device_manager.py에서 연결 흐름 추적

**API 오류:**
- 해당 라우터 파일 확인 (backend/app/routers/)
- 서비스 계층 로직 추적
- API 직접 호출: `curl -s http://localhost:8000/api/<endpoint>`

**WebSocket 문제:**
- backend/app/main.py의 /ws/screen, /ws/playback 핸들러
- frontend/src/context/DeviceContext.tsx의 WebSocket 연결 로직
- frontend/src/services/websocket.ts

**이미지 비교:**
- backend/app/services/image_compare_service.py
- CompareMode 설정 (FULL, SINGLE_CROP, FULL_EXCLUDE, MULTI_CROP)
- similarity_threshold 값

### 4단계: 진단 결과

```
## 진단 결과
- 증상: [분류된 증상]
- 원인: [근본 원인]
- 해결: [구체적 수정 방법]
- 예방: [재발 방지 방안]
```

수정이 필요하면 `/auto`로 자동 수정을 안내합니다.

## 규칙
- 코드를 수정하지 않습니다 — 진단 결과만 보고합니다
- 수정이 필요하면 `/auto`를 안내합니다
- 로그 메시지가 포함되어 있으면 해당 코드 위치를 정확히 추적합니다
